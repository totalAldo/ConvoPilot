Privacy Policy

ConvoPilot is designed to keep conversation analysis on device by default.

This policy explains what ConvoPilot processes, what it stores, and the controls users have over transcription, summaries, notifications, and optional location-based legal-region suggestions.

Effective Date March 27, 2026
Default Processing Mode On-device transcription
Contact convopilot@totalaldo.com
Raw audio

Not saved by default and intended to stay in memory only during a live session.

Summaries

Stored on device only if the user explicitly enables summary saving.

Cloud processing

Off by default and should require explicit user opt-in when available.

1. What We Process

Information used by the app

  • Microphone audio during a user-started session
  • Generated transcript text and timestamps
  • Derived conversation signals such as sentiment, mood shifts, highlights, and optional coaching cues
  • User settings such as appearance, analysis preferences, legal region, and privacy preferences
  • Optional on-device session summaries if saving is enabled
  • Optional approximate legal-region suggestion inputs if the user requests location help
2. What We Do Not Claim

Important limits

  • ConvoPilot is not intended to diagnose health conditions.
  • ConvoPilot is not a therapy or emergency service.
  • ConvoPilot does not provide legal advice.
  • ConvoPilot version 1 does not guarantee universal two-sided call capture on iPhone.
3. On-Device Processing

Default privacy posture

Transcription

ConvoPilot is designed to use on-device Apple speech technologies by default whenever supported on the current device and locale.

Realtime analysis

Sentiment, emotional-state cues, highlights, live feedback, and optional coaching cues are intended to be generated on device from the current transcript window.

Storage

Raw audio is not intended to be retained. Session summaries are only stored locally if the user enables summary saving.

4. Cloud Processing

If cloud transcription is ever activated

Current default Cloud transcription is off by default. Users should be warned before enabling it.

If a future or supported configuration enables cloud transcription, ConvoPilot should provide a clear warning, require explicit opt-in, and show a visible cloud indicator when cloud processing is active for a session.

If cloud processing becomes active in production, this privacy policy should be updated before that version is submitted or released.

5. Permissions

Why permissions are requested

  • Microphone: to capture audio during a live session.
  • Speech recognition: to transcribe speech.
  • Notifications: to show optional local analysis prompts.
  • Location: only if the user asks the app to suggest a legal region.
6. User Controls

What users can control

  • Whether optional coaching suggestions are enabled
  • Whether local notifications are allowed
  • Whether session summaries are stored on device
  • Whether cloud transcription is allowed when such a mode exists
  • What legal region is selected for general guidance
7. Sharing and Sale of Data

How information is handled

ConvoPilot is not designed to sell personal information or use transcript data for advertising.

By default, transcript and analysis data are intended to remain on device. If cloud transcription is ever activated, the related privacy flow should describe the external processing clearly before the user starts the session.

8. Children

Children's privacy

ConvoPilot is not directed to children. Do not use the app in a way that violates applicable laws or school, workplace, or guardian policies.

9. Changes

Policy updates

This policy may change if the app's processing model, storage behavior, or external services change. The effective date above will be updated when material changes are made.
